# Roadmap: «Холодная плазма — Северодвинск»

## Текущий статус

Проект находится на этапе рабочего fullstack-прототипа. Уже есть Go API, React SPA, PostgreSQL, Docker Compose, миграции, личный кабинет, онлайн-запись, AI-чат, email-подтверждение, восстановление пароля, PDF-памятка, админская страница и Telegram/SMS-контур уведомлений.

Реализовано на текущем этапе:

- регистрация требует российский мобильный номер;
- телефон нормализуется к формату `+7XXXXXXXXXX`;
- один номер телефона нельзя привязать к двум аккаунтам;
- запись запрещена без `phone_verified=true`;
- подтверждение телефона работает через SMS-код;
- запись поддерживает выбор процедуры, сообщение и несколько вариантов даты/времени;
- бонусы больше не начисляются автоматически за запись;
- пользователь смотрит баланс бонусов в личном кабинете;
- админ может начислять бонусы пользователю по номеру телефона;
- админ управляет каналами уведомлений о новых записях: Telegram, SMS или оба;
- тип входящего лида определяется внутри backend: `notify_admin`, `cold`, `warm`, `hot`;
- Telegram-бот сделан в трёхслойной архитектуре и отправляет уведомления о новых записях;
- добавлена заготовка входа через VK ID;
- email для VK-пользователей может быть пустым;
- `/admin` скрыт от неадминистраторов как обычная 404-страница.

## Этап 1. Стабилизация текущего MVP

- Применить миграции:
  - `004_phone_vk_auth`;
  - `005_booking_password_telegram`;
  - `006_admin_notification_settings`.
- Проверить регистрацию с телефонами:
  - `+79001234567`;
  - `89001234567`;
  - `9001234567`;
  - не российский номер.
- Проверить, что один телефон нельзя сохранить на двух аккаунтах.
- Проверить подтверждение телефона через `SMS_PROVIDER=log`.
- Проверить, что пользователь без подтверждённого телефона не может создать запись.
- Проверить новую запись с несколькими вариантами даты.
- Проверить, что за запись не начисляются бонусы.
- Проверить, что админ может начислить бонусы по номеру телефона.
- Проверить, что пользователь видит баланс и историю бонусов в кабинете.

## Этап 2. Настройка SMS

### Dev-режим

- Оставить:
  - `SMS_PROVIDER=log`;
  - `SMS_HTTP_URL=`;
  - `SMS_HTTP_TOKEN=`;
  - `SMS_FROM=cold_plasma`.
- Отправить код подтверждения телефона из кабинета.
- Смотреть код в логах:

```bash
docker compose logs -f api
```

Ожидаемый формат:

```text
phone verification code for +79001234567: 123456
```

### Production SMS

- Выбрать SMS-провайдера по РФ.
- Решить формат интеграции:
  - использовать текущий generic HTTP adapter;
  - или написать отдельную реализацию `sms.Sender` под конкретный API.
- Заполнить:
  - `SMS_PROVIDER=http`;
  - `SMS_HTTP_URL`;
  - `SMS_HTTP_TOKEN`;
  - `SMS_FROM`.
- Проверить сценарии:
  - SMS-код для подтверждения телефона;
  - SMS-уведомление админу о новой записи, если включено в `/admin`;
  - ошибка провайдера;
  - пустой/неверный номер;
  - повторная отправка кода.
- Добавить rate limit на отправку SMS-кодов.
- Добавить хранение/просмотр статусов отправки SMS, если провайдер поддерживает callback.

## Этап 3. Telegram-бот

- Создать бота через BotFather.
- Заполнить:
  - `TELEGRAM_BOT_TOKEN`;
  - `TELEGRAM_ADMIN_CHAT_ID`.
- В `/admin` включить Telegram-уведомления.
- Проверить уведомление о новой записи.
- Проверить тип лида в сообщении:
  - `notify_admin`;
  - `cold`;
  - `warm`;
  - `hot`.
- Доработать reminder за день до записи.
- Расширить bot-сервис будущими командами:
  - список ближайших записей;
  - изменение статуса записи;
  - быстрый контакт с клиентом;
  - ручное начисление бонусов из Telegram.

## Этап 4. Настройка VK ID

- Создать приложение VK ID.
- Получить:
  - `VK_CLIENT_ID`;
  - `VK_CLIENT_SECRET`.
- Добавить redirect URL:
  - Docker: `http://localhost:8081/account`;
  - Vite dev: `http://localhost:5173/account`.
- Заполнить `.env`:
  - `VK_CLIENT_ID`;
  - `VK_CLIENT_SECRET`;
  - `VK_REDIRECT_URI`;
  - `VITE_VK_CLIENT_ID`;
  - `VITE_VK_REDIRECT_URI`.
- Пересобрать frontend через `make up`.
- Проверить, что VK возвращает российский телефон и backend создаёт пользователя с `phone_verified=true`.

## Этап 5. UX и контент

- Доработать кабинет:
  - отдельный статус email и телефона;
  - понятный сценарий повторной отправки SMS;
  - подсказка для пользователей VK.
- Доработать страницу записи:
  - защита от выбора прошедшего времени;
  - понятный список выбранных дат;
  - пояснение, что дата будет подтверждаться администратором.
- Доработать админку:
  - список заявок/записей;
  - фильтр по типу лида;
  - история начисления бонусов по клиенту;
  - редактирование статуса записи.
- Уточнить тексты AI-консультанта:
  - не здороваться повторно в продолжении диалога;
  - не предлагать запись слишком рано;
  - предлагать запись только при явном интересе пользователя.
- Добавить реальные фото, FAQ, адрес, схему прохода и парковку.

## Этап 6. Тестирование

- Backend:
  - регистрация с российскими/не российскими номерами;
  - запрет повторной привязки телефона;
  - booking guard по `phone_verified`;
  - phone verification flow;
  - новая запись с несколькими датами;
  - отсутствие начисления бонусов за запись;
  - админское начисление бонусов по телефону;
  - VK happy path и отказ без российского телефона.
- Frontend:
  - сборка `npm run build`;
  - регистрация и подтверждение телефона;
  - запись после подтверждения;
  - админские переключатели Telegram/SMS;
  - начисление бонусов админом;
  - VK-вход после настройки VK ID.
- E2E:
  - регистрация → SMS-код из логов → запись;
  - запись → уведомление админу;
  - админ → начисление бонусов → пользователь видит баланс;
  - обычный пользователь не видит `/admin`.
